四、hexo静态博客框架搭建

4.1 检查安装并优化环境

  1. 在Git窗口输入node -v查看node版本信息来确保node已安装成功；
  2. 在Git窗口输入npm -v查看npm版本信息来确保npm已安装成功；这里顺便介绍npm，npm是随同NodeJS一起安装的包管理工具，能解决NodeJS代码部署上的很多问题，具体介绍去问度娘。
  3. 操作如下图：
  4. 如果npm不是最新的可以通过npm install -g npm指令更新；
  6. 由于使用自带的npm，下载速度会很慢，这里建议先更改npm仓库地址，使用cnpm。cnpm是由阿里的淘宝团队在国内制作的一个跟npm一样的镜像，两者包的同步频率为十分钟一次。这里有两种方式，推荐使用第一种。
  • 通过改变地址来使用淘宝镜像
    a.在Git窗口输入npm config get registry查看当前npm的仓库地址
    一般npm的默认仓库地址为：https://registry.npmjs.org/
    b.cnpm的仓库地址为：https://registry.npm.taobao.org/
    在Git窗口输入npm config set registry https://registry.npm.taobao.org将默认下载地址改变cnpm的地址，从而达到不安装cnpm就能采用淘宝镜像的目的，然后使用上面的get命令查看是否设置成功。
  • 安装CNPM
    a.在Git窗口输入npm install -g cnpm --registry=https://registry.npm.taobao.org
    b.安装后，使用以下命令cnpm -v测试是否安装成功。成功后，以后都使用 cnpm 代替以前 npm 来执行命令！

    4.2 安装hexo博客框架

  1. 在Git窗口输入cnpm install -g hexo-cli全局安装hexo；

  2. 安装完成后，可以在Git窗口输入hexo -v，来验证是否正确安装；

    4.3 搭建本地博客

  3. 选择一个磁盘作为博客文件的存放位置，然后新建一个文件夹，比如名为myblog的文件夹，创建完成后，先不要点进去，在空白处点击鼠标右键，选择Git Bash Here，打开Git。

  4. 在Git窗口输入hexo init myblog来初始化文件夹；

  5. 在Git窗口输入cd myblog来进入myblog文件夹；

  6. 在Git窗口输入cnpm install安装本地博客所需要的依赖文件；等待运行完成，此时文件夹里会多了很多文件，此时Hexo框架的本地搭建已经完成了。
    注意：后续的命令均需要在站点目录下（即文件夹内）使用Git Bash运行

  7. 在Git窗口输入hexo generate或者hexo g

  8. 在Git窗口输入hexo server或者hexo s

  9. 浏览器中打开 http://locakhost:4000，可以看到一个网页，说明Hexo博客已经成功在本地运行。

    七、常用命令

每次修改完本地的文件后要执行以下命令，重新部署到github上：
hexo clean——清除多余的文件
hexo generate或者hexo g——生成文件
hexo deploy或者hexo d——将本地博客部署到服务器上，比如Github

常用命令
hexo new “postName” #新建文章
hexo new page “pageName” #新建页面
hexo generate #生成静态页面至public目录
hexo server #开启预览访问端口（默认端口4000，’ctrl + c’关闭server）
hexo generate或者hexo d #
hexo deploy #将.deploy目录部署到GitHub
hexo help # 查看帮助
hexo version #查看Hexo的版本

八、报错总结

1.出现 error deployer not found:git 或者 error deployer not found:github 的错误
解决办法：执行 npm install hexo-deployer-git –save

2.出现类似 FATAL can not read a block mapping entry; a multiline key may not be an implicit key at line 70, column 1: … 错误
解决办法：_config.yml中配置项的冒号后面要用空格隔开，再跟内容

3.中文乱码问题
解决办法：将文件的内容编码改为UTF8格式

4.’hexo sever’能够成功运行，但是localhost:4000无法访问
解决办法：执行hexo s -p 5000，改用其他端口启动
